⬡ BONE FORGE — Help & Instructions

Quick Start — From Zero to Animation

1
Load sprites — Click + Sprites in the topbar. Load separate PNG files for each body part (head, torso, upper arm, lower arm, etc.). Each part should have a transparent background. The editor auto-trims empty space.
2
Add root bone — Click + root in the Skeleton panel. Then press B (Bone mode) and click+drag in the viewport to draw the first bone (e.g. torso). The drag sets length and direction.
3
Build the skeleton — Select a bone, press B, click+drag from its tip to add a child bone. Repeat for each limb. You can also drag bones in the Skeleton tree to re-parent them.
4
Attach sprites — Select a bone in the tree, then click a sprite in the Sprites panel to attach it. A green dot on the bone means it has a sprite.
5
Set attachment point — In Properties, click 🖊 Set on sprite…. Drag the cyan ⊕ START handle to where the bone attaches (e.g. shoulder joint on an arm sprite). Drag yellow ◎ END to set bone length. Click Confirm. The sprite will not rotate or move.
6
Z-order — Use the Z-Order panel to control which sprites render on top. Use ↑↓ buttons to reorder.
7
Create an animation — Click + in the Animations panel (right side). Give it a name and frame count.
8
Animate — Navigate to a frame with ◀ ▶ or ← → keys. In Select mode, drag a bone tip to rotate it. Then click + Key to save a keyframe, or + Pose to save all bones at once. Repeat at different frames.
9
Preview — Press ▶ Play or Space to preview. Use 🧅 Onion to see ghost frames (blue = previous, red = next).
10
Export — Click ⬇ Export ZIP to download individual PNG frames. Or 💾 Save to save the whole project as JSON for later editing.

Viewport Controls

  • Drag bone tip — Rotate the bone around its attachment point
  • Drag bone root (root bones only) — Move the bone in world space
  • Right-click drag / Alt+drag / Middle-click — Pan the view
  • Scroll — Pan up/down/left/right
  • Shift+Scroll / Ctrl+Scroll / Pinch — Zoom in/out
  • I IK mode — Drag any bone tip and the whole parent chain follows (FABRIK inverse kinematics)

Keyboard Shortcuts

  • V — Select mode
  • B — Add Bone mode
  • I — IK mode
  • P — Pan mode
  • O — Toggle Onion Skinning
  • Space — Play / Stop
  • ← → — Previous / Next frame
  • Delete — Delete selected bone (and its children)
  • Ctrl+Z — Undo
  • Ctrl+Y / Ctrl+Shift+Z — Redo
  • Ctrl+C — Copy current pose
  • Ctrl+V — Paste pose at current frame

Timeline

  • Click on a frame — Seek to that frame
  • Click a keyframe diamond — Select that bone + seek to frame
  • Shift+Click keyframes — Multi-select keyframes
  • Drag selected keyframes — Move them in time together
  • + Key — Add keyframe for selected bone at current frame
  • + Pose — Add keyframes for ALL bones at current frame
  • − Key — Delete keyframe for selected bone at current frame
  • TL Zoom slider — Zoom the timeline horizontally

Skeleton Tree

  • Click — Select bone
  • Double-click — Rename bone inline
  • Drag & Drop — Re-parent a bone to another
  • Green dot = sprite attached, yellow dot = no sprite

Animations Panel

  • Click — Switch to that animation
  • Double-click name — Rename animation inline
  • Thumbnail shows frame 0 pose
  • Update Name/Frames/Loop with the fields below, then click Update

Sprite Inspector (Set on sprite…)

  • Drag ⊕ START — attachment point on the sprite (where the bone "plugs in")
  • Drag ◎ END — sets bone length (how long the bone is in sprite pixels)
  • START must stay inside the sprite; END can go outside for long bones
  • Click Reset START to center to reset pivot to sprite center
  • Bone angle is NOT changed — sprite stays upright after Confirm
  • Rotate bones in the viewport after confirming

Tips for Pixel Art

  • Prepare each body part as a separate transparent PNG
  • Name files clearly: body.png, arm_upper.png, etc.
  • Use Z-Order panel to set which parts are in front (e.g. front arm above torso)
  • Use 🧅 Onion to check timing — blue = where you were, red = where you're going
  • Use + Pose at frame 0 to lock the rest pose, then animate from there
  • Save your project with 💾 Save often — the JSON includes all sprites as base64
